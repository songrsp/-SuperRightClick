# 团队分发

目标：让团队同事**双击就能装、能用**，而不用各自装 Xcode、各自签名。

## 方案对比

| 方案 | 适用 | 要点 |
|------|------|------|
| A. Developer ID + 公证 + DMG | 推荐，对外/跨机最稳 | 需付费账号；过 Gatekeeper，任何 Mac 双击可用 |
| B. 内部自签 + 手动信任 | 纯内网、机器可控 | 免公证，但每台机要手动允许 |
| C. 直接发 Xcode 工程 | 全是开发同事 | 各自 `xcodegen + ⌘R`，见构建文档 |

下面主要讲 **方案 A**。

## 方案 A：Developer ID 签名 + 公证

### 1. 用 Developer ID 签名构建

确保 `Local.xcconfig` 里 `DEVELOPMENT_TEAM` 是付费团队，签名证书选 **Developer ID Application**。
扩展 `.appex` 会随 App 一起被签名（嵌在 `Contents/PlugIns` 内）。

```bash
xcodegen generate
xcodebuild -project SuperRightClick.xcodeproj -scheme SuperRightClick \
           -configuration Release -derivedDataPath build build
APP="build/Build/Products/Release/SuperRightClick.app"
```

> 因为主程序非沙盒，必须开启 **Hardened Runtime**（`project.yml` 已设 `ENABLE_HARDENED_RUNTIME: YES`），否则公证不过。

### 2. 验证签名

```bash
codesign --verify --deep --strict --verbose=2 "$APP"
# 确认扩展也签了
codesign -dv --verbose=4 "$APP/Contents/PlugIns/FinderExtension.appex"
```

### 3. 打包并公证

```bash
# 打成 zip 交给公证
ditto -c -k --keepParent "$APP" SuperRightClick.zip

# 用 App 专用密码或 API Key 提交公证（凭据需提前 store-credentials）
xcrun notarytool submit SuperRightClick.zip \
      --keychain-profile "AC_NOTARY" --wait

# 公证通过后装订票据
xcrun stapler staple "$APP"
```

> `--keychain-profile` 需先执行一次：
> `xcrun notarytool store-credentials AC_NOTARY --apple-id you@xx.com --team-id ABCDE12345 --password <App专用密码>`

### 4. 做成 DMG 发出去

```bash
mkdir dmg && cp -R "$APP" dmg/
ln -s /Applications dmg/Applications
hdiutil create -volname "超级右键" -srcfolder dmg -ov -format UDZO SuperRightClick.dmg
rm -rf dmg
```

把 `SuperRightClick.dmg` 发给同事，拖进 Applications 即可。

## 同事首次使用要点（写进发布说明）

1. 把 App 拖进「应用程序」，打开一次。
2. 设置窗口 **第 1 步** → 启用「超级右键扩展」。
3. **第 2 步** → 在「完全磁盘访问权限」里加入本 App 并打开（**卸载/清理功能必须**），然后完全退出再重开。
4. 建议把 App 加入登录项（第 3 步），开机自启常驻后台。

## 升级

自带自动更新不在 v1 范围内。简单做法：每次发新 DMG 让同事覆盖安装。要做自动更新可接入 [Sparkle](https://sparkle-project.org/)。
