# 构建与签名

## 一、前置条件

- macOS 13 或更高
- Xcode 15+（含命令行工具：`xcode-select --install`）
- [XcodeGen](https://github.com/yonyz/XcodeGen)：`brew install xcodegen`
- 一个 Apple 开发者账号
  - **免费个人账号**：可本机自签调试，签名 7 天过期，不能分发
  - **付费账号（$99/年）**：可生成 Developer ID，公证后团队分发

## 二、生成并打开工程

```bash
cd SuperRightClick
cp Local.xcconfig.example Local.xcconfig
# 必改：打开 Local.xcconfig，把 DEVELOPMENT_TEAM 换成你的 Team ID
open Local.xcconfig
./scripts/bootstrap.sh
open SuperRightClick.xcodeproj
```

> `.xcodeproj` 是由 `project.yml` 生成的产物，**不要手改、也不必提交到 Git**。改工程配置请改 `project.yml` 后重新 `xcodegen generate`。

## 三、关于 Team ID / Bundle ID / App Group

这三者必须前后一致，否则 App Group 信箱不通、扩展加载不出来。需要同步的位置：

| 项目 | 位置 |
|------|------|
| Team ID | `Local.xcconfig` 的 `DEVELOPMENT_TEAM`（不入库） |
| App Bundle ID | `project.yml` → `com.team.superrightclick`（可改成你司域名倒写） |
| 扩展 Bundle ID | 必须是 `<App Bundle ID>.FinderExtension` |
| App Group | `Shared/AppConfig.swift` + `App/App.entitlements` + `FinderExtension/FinderExtension.entitlements` 三处一致 |

> 改 Bundle ID / App Group 时，用编辑器全局搜索 `com.team.superrightclick` 与 `group.com.team.superrightclick` 一并替换最省事。

App Group 需要在 Apple 开发者后台或 Xcode 的 Signing & Capabilities 里注册。Xcode 自动签名通常会帮你创建。

## 四、命令行构建（CI 可用）

```bash
xcodegen generate
xcodebuild -project SuperRightClick.xcodeproj \
           -scheme SuperRightClick \
           -configuration Release \
           -derivedDataPath build \
           build
# 产物：build/Build/Products/Release/SuperRightClick.app
```

## 五、常见构建问题

**Q：右键看不到菜单？**
1. 设置窗口里确认"第 1 步"显示已启用；没有就点"打开扩展设置"勾选。
2. 终端执行 `pluginkit -m | grep superrightclick` 看扩展是否注册。
3. 重置扩展缓存：`pluginkit -e use -i com.team.superrightclick.FinderExtension`，再 `killall Finder`。

**Q：卸载/清理时报"没权限删除"？**
→ 主程序需要「完全磁盘访问权限」。设置窗口"第 2 步"打开隐私设置，把 `SuperRightClick.app` 加进去并打开开关，然后**完全退出再重开**主程序。

**Q：扩展加载了但点了没反应？**
→ 多半是主程序没在后台跑或 App Group 不通。打开 Console.app，按 subsystem `com.team.superrightclick` 过滤看日志。确认两个 target 的 App Group、Team 完全一致。

**Q：免费账号签名 7 天后失效？**
→ 正常现象。重新 ⌘R 跑一次即可续签。要长期用请上付费账号 + Developer ID（见团队分发文档）。
