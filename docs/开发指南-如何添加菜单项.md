# 开发指南：加一个新的右键功能

以"**用十六进制编辑器打开**"为例，演示三步加一个菜单项。

## 第 1 步：声明动作

`Shared/Command.swift` 的 `ActionID` 枚举里加一个 case：

```swift
enum ActionID: String, Codable, CaseIterable {
    // ...已有...
    case openInHexEditor   // 新增
}
```

## 第 2 步：加菜单项

`FinderExtension/MenuBuilder.swift`，在合适的分组里加一行。例如放进"打开方式"子菜单：

```swift
submenu("打开方式", [
    item("在此打开终端", .openInTerminal),
    item("用 VS Code 打开", .openInVSCode),
    item("用十六进制编辑器打开", .openInHexEditor),   // 新增
])
```

如果只想在选中文件时出现，把它放进 `if hasSelection { ... }` 块里即可。

## 第 3 步：实现执行逻辑

`App/CommandExecutor.swift` 的 `switch command.action` 里加分支：

```swift
case .openInHexEditor:
    let r = Shell.run("/usr/bin/open", ["-a", "Hex Fiend"] + urls.map { $0.path })
    if r.status != 0 { Alerts.warn("未找到 Hex Fiend", "请先安装。") }
```

重新 ⌘R 运行即可生效（改了扩展的话，Finder 会自动重载扩展；必要时 `killall Finder`）。

## 速查：常用 API

**拿到用户选中的文件**（扩展里）：
```swift
let urls = FIFinderSyncController.default().selectedItemURLs() ?? []
let folder = FIFinderSyncController.default().targetedURL()   // 右键空白处时的当前文件夹
```

**就地操作 vs 投递主程序**：
- 只动剪贴板 → 在 `FinderSyncController.handleMenu` 里直接做（参考 `copyPath`），最快、不抢焦点，**不要**唤起主程序。
- 要删文件 / 跑命令 / 弹窗 → 走 `Mailbox.send(command)` + `AppLauncher.wakeContainerApp()`，由主程序 `CommandExecutor` 执行。

**带参数的动作**（如「用某 App 打开」需要传 App 路径）：
菜单项载荷是字典 `{"action": <raw>, "arg": <值>}`。建项时 `item("标题", .yourAction, arg: "参数")`，主程序里用 `command.arg` 取。参考 `openWith`。

**给功能加一个分组开关**：在 `Shared/Preferences.swift` 加一个 `Key` 与访问器，`MenuBuilder` 里用 `if Pref.xxx { ... }` 包住，`SettingsView` 里加一个 `@AppStorage(Pref.Key.xxx, store: Pref.store)` 的 `Toggle`。三处共用 App Group，改完下次右键即生效。

**跑子进程**（主程序里）：
```swift
Shell.run("/usr/bin/zip", ["-r", "out.zip", "foo"], cwd: someFolder)
Shell.osascript("tell application \"Finder\" to activate")
```

**弹窗**：
```swift
Alerts.info("标题", "内容")
Alerts.warn("标题", "内容")
```

**开个 SwiftUI 窗口**：参考 `BatchRenameWindow.show(for:)` —— 用 `NSHostingController` 包 SwiftUI 视图，塞进 `NSWindow`，并用 `static` 变量持有引用防止被释放。

## 调试技巧

- 日志：`Log.info("...")` 走统一日志，Console.app 按 subsystem `com.team.superrightclick` 过滤；`Log.debugFile("...")` 落到 App Group 的 `debug.log`（扩展沙盒写不了 `/tmp`，所以用共享容器），看：`tail -f ~/Library/Group\ Containers/group.com.team.superrightclick/debug.log`。
- 扩展不刷新：`killall Finder`；仍不行 `pluginkit -e use -i com.team.superrightclick.FinderExtension`。
- 信箱里看积压指令：
  ```bash
  ls "~/Library/Group Containers/group.com.team.superrightclick/mailbox/"
  ```
- 主程序没反应：多半没在后台跑 / App Group 不一致，先确认设置窗口三步都 OK。
