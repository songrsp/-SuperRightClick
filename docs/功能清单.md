# 功能清单

菜单结构（右键 → **⚡️ 超级右键**）。带 ★ 的为选中文件后才出现，带 ◆ 的为选中 `.app` 才出现。

## 打开方式
| 菜单项 | 动作 | 说明 |
|--------|------|------|
| 在此打开终端 | `openInTerminal` | 在当前文件夹打开 Terminal |
| 用 VS Code 打开 | `openInVSCode` | 用 Visual Studio Code 打开选中项/当前目录 |
| 用 〈某 App〉打开（动态） | `openWith` | 用 `NSWorkspace` 查询能打开选中文件的全部 App，动态列出（去重、限 8 个）。App 路径通过 `Command.arg` 传给主程序 |

## 复制信息 ★
| 菜单项 | 动作 | 说明 |
|--------|------|------|
| 复制路径 | `copyPath` | 多选时每行一个绝对路径（扩展内直接完成） |
| 复制文件名 | `copyName` | 复制文件名（含扩展名） |
| 复制所在目录 | `copyDirPath` | 复制文件所在文件夹路径 |

## 新建
| 菜单项 | 动作 | 说明 |
|--------|------|------|
| 文本文件 (.txt) | `newFileTxt` | 在当前目录新建并自动避免重名 |
| Markdown (.md) | `newFileMarkdown` | 同上 |
| 文件夹 | `newFolder` | 新建文件夹 |

## 文件管理 ★
| 菜单项 | 动作 | 说明 |
|--------|------|------|
| 复制到… | `copyToFolder` | 弹文件夹选择器，复制过去 |
| 移动到… | `moveToFolder` | 弹文件夹选择器，移动过去 |
| 压缩为 ZIP | `compress` | 单文件用 `ditto`，多文件用 `zip` |
| 解压到此处 | `decompress` | 仅选中压缩包时出现，zip/tar/gz 自动识别 |
| 批量重命名… | `batchRename` | 多选时出现，前缀+序号，带预览 |
| 移到废纸篓 | `moveToTrash` | 安全删除 |

## 开发者 ★
| 菜单项 | 动作 | 说明 |
|--------|------|------|
| 复制 SHA256 | `copyHashSHA256` | 流式计算，大文件不爆内存 |
| 复制 MD5 | `copyHashMD5` | 同上 |

## 仿 Windows 常驻
| 菜单项 | 动作 | 说明 |
|--------|------|------|
| 显示/隐藏隐藏文件 | `toggleHiddenFiles` | 切换 `AppleShowAllFiles` 并重启 Finder |
| 刷新 | `refreshFinder` | `killall Finder` |
| 属性 ★ | `showProperties` | 打开 Finder「显示简介」窗口 |

## 卸载 ◆
| 菜单项 | 动作 | 说明 |
|--------|------|------|
| 彻底卸载此应用并清理残余… | `uninstallApp` | 读取 Bundle ID → 扫描 `~/Library` 等目录的关联文件 → 弹窗逐项确认 → 移到废纸篓 |

### 残余扫描覆盖目录
`~/Library` 下：Application Support、Caches、Preferences、Logs、Containers、Group Containers、Saved Application State、HTTPStorages、WebKit、LaunchAgents、Cookies；
`/Library` 下：Application Support、Caches、Preferences、LaunchAgents、LaunchDaemons。

匹配规则：文件/文件夹名包含该 App 的 **Bundle ID** 或 **应用名**（忽略空格大小写）。

## 功能开关（设置页）
设置窗口里可逐组开关菜单（存在 App Group 共享 `UserDefaults`，扩展下次出菜单即生效）：打开方式 / 复制信息 / 新建 / 文件操作 / 开发者 / 仿 Windows / 卸载。
另有「收进子菜单 vs 平铺」开关：默认平铺到 Finder 原生右键；打开后所有项收进一个「⚡️ 超级右键」子菜单。
