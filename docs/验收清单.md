# 验收清单 & 排雷点

> 脚手架代码已做过**静态自检**（20 个 ActionID 在「声明/菜单/执行」三处对齐、Bundle ID/App Group/主类名一致），但**未在真机 Xcode 编译过**。下面按"先编译、再跑通、后打磨"列出要做的事，并标出最可能踩坑的地方。

## A. 编译期排雷（大概率要处理）

1. **FinderSync 框架链接**
   `project.yml` 没有显式链接 `FinderSync.framework`，依赖 Swift 自动链接。若出现 `Undefined symbol` / `module not found`，在两个 target 的 `dependencies` 里加：
   ```yaml
   dependencies:
     - sdk: FinderSync.framework
   ```
   主程序 target 也用到了 `import FinderSync`（设置页查询扩展状态），同样需要。

2. **App 扩展产物类型与嵌入**
   确认 `FinderExtension` 以 `com.apple.product-type.app-extension` 产出 `.appex`，并被嵌入到 `SuperRightClick.app/Contents/PlugIns/`。XcodeGen 的 `type: app-extension` + `embed: true` 应已处理；编译后用
   `ls SuperRightClick.app/Contents/PlugIns/` 核实。

3. **签名 & App Group 注册**
   复制 `Local.xcconfig.example` 为 `Local.xcconfig`，把 `DEVELOPMENT_TEAM` 改成真实 Team ID。App Group `group.com.team.superrightclick` 需在开发者后台/Xcode Signing&Capabilities 注册（自动签名一般会创建）。两个 target 都要勾上同一个 App Group。

4. **Swift 并发模式**
   代码按 **Swift 5 语言模式**（`SWIFT_VERSION: 5.0`）编写，`@MainActor` 跨隔离调用为宽松检查。**若你切到 Swift 6 严格并发**，需要修：`CommandExecutor`(@MainActor)、`UninstallManager`(@MainActor) 与后台 `DispatchQueue.global` 的隔离边界（已对纯 IO 的 `scanResiduals/size/bundleID` 标了 `nonisolated`）。

5. **部署目标**
   `ForEach($binding)`、`NSHostingController`、`CryptoKit` 等需 macOS 13+。`project.yml` 已设 13.0，别调低。

## B. 运行期排雷（编译过后重点验证）

6. **扩展 → 主程序的唤起**（最需要真机验证的一处）
   扩展用 `NSWorkspace.shared.openApplication` 唤起主程序。沙盒扩展能否唤起容器 App 在不同系统版本上表现不一。**保险机制已内置**：主程序每 0.5s 轮询信箱，所以**只要主程序在后台运行，即使唤起被拦也照样执行**。验证步骤：先手动打开主程序 → Finder 右键点"压缩 ZIP" → 看是否生成 zip。若唤起不灵，启用兜底 URL Scheme（`superrightclick://`，`URLCommandParser` 已实现）或确保主程序常驻（登录项）。

7. **菜单是否出现**
   `menu(for:)` 返回的是"一个父项 + 子菜单"。若 Finder 里看不到菜单：先确认扩展已在「系统设置→登录项与扩展→访达扩展」勾选；再 `killall Finder`；仍不行可改成把菜单项直接加到根 `NSMenu`（去掉外层父项）对比。

8. **完全磁盘访问权限**
   卸载/清残余、读 `~/Library` 需要给**主程序**授予「完全磁盘访问权限」，授权后要**完全退出再重开**主程序才生效。

9. **自动化（Automation）权限**
   `showProperties` 和"刷新"会用 `osascript` 控制 Finder，首次会弹"允许控制 Finder"。需在「隐私与安全性→自动化」放行。

10. **打开终端/VS Code**
    `open -a Terminal` / `open -a "Visual Studio Code"`。若用户装的是 iTerm 或别的编辑器，在 `CommandExecutor.openTerminal/openVSCode` 里改应用名，或做成设置项。
